# Spec 2 — Config link on the invoice detail page

**Follow-up.** The reverse direction of the relationship: from an invoice, link
back to the recurring config that generated it.

## Depends on

- Spec 0 (the `recurring_invoice_id` value must exist on the invoice).

## Backend changes

- Invoice detail response includes `recurringInvoiceId` (already on the entity
  after Spec 0; expose it in the responder / response DTO).
- *(Optional)* To display the config's human id (`REC-…`) or schedule on the
  invoice page, either join it server-side or have the frontend fetch the
  config by id.

## Frontend changes

- On the invoice detail page, when `recurringInvoiceId` is set, render
  "Generated by recurring schedule REC-…" linking to
  `/recurring-invoices/:id`.
- Hidden entirely for manually-created invoices (null `recurringInvoiceId`).

## Acceptance criteria

- A generated invoice shows a working link back to its config.
- A manual invoice shows nothing.

## Tests

- Responder/DTO test: `recurringInvoiceId` is present in the response when set,
  absent/null otherwise.
